# Intelligent Monitor - Debug Logs Directory

Эта директория содержит детальные логи от Intelligent Monitor - системы интеллектуального мониторинга проекта AILLM.

## Файлы логов

- **monitor.log** - Основной файл логов мониторинга (все события DEBUG, WARNING, ERROR, CRITICAL)
- **critical_issues.log** - Критические проблемы системы
- **monitor_state.json** - Состояние мониторинга (обновляется каждую минуту)

## Что мониторится

### Системные ресурсы
- Использование CPU (%)
- Использование памяти (MB, %)
- Наличие и использование GPU

### Компоненты системы
- Engine (основной движок)
- LLM Manager (менеджер провайдеров)
- Vector Store (векторное хранилище)
- Context Manager (менеджер контекста)
- Agent Registry (реестр агентов)
- Orchestrator (оркестратор)
- Tool Registry (реестр инструментов)
- Memory (долгосрочная память)

### Метрики производительности
- Время выполнения задач
- Время ответа LLM
- Частота ошибок
- Аномалии в поведении

## Уровни серьезности проблем

- **INFO** - Информационные сообщения
- **WARNING** - Предупреждения (высокое использование ресурсов, аномалии)
- **ERROR** - Ошибки выполнения
- **CRITICAL** - Критические проблемы (необработанные исключения, сбои компонентов)

## API эндпоинты

### Получить отчет о здоровье системы
```
GET /api/v1/monitoring/health
```

### Получить информацию о debug логах
```
GET /api/v1/monitoring/debug-logs
```

## Автоматические действия

Монитор автоматически:
- Обнаруживает проблемы и аномалии
- Логирует детальную информацию в LOGS_DEBUG
- Отслеживает производительность компонентов
- Сохраняет состояние каждую минуту
- Ротация логов (100 MB, 30 дней хранения)

## Настройка

Монитор запускается автоматически при инициализации Engine. Интервал проверки: 5 секунд.

Для изменения настроек см. `backend/core/intelligent_monitor.py`
